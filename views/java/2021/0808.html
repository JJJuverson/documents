              # 对象和引用

              • 对象是通过new出来的，在堆内存中储存
              • 但凡是变量，并且该变量中保存了内存地址，且指向该内存地址指向的堆内存中的对象叫引用。

              # 访问实例对象

              • 引用.实例变量名
              • 通过=赋值的方式修改实例变量的值

              # 空指针异常

              • 该对象没有引用指向它了，java中规定了堆内存程序员是无权直接操作的，此时这个对象 飘走了，该对象被垃圾回收机制回收。
              • 垃圾回收机制，主要回收的是堆内存当中的垃圾数据，当没有任何引用指向该对象时，这个对象就会被垃圾 回收机制回收。

              # 规定

              • java中规定，参数传递的时候和类型无关，不管是基本数据类型，还是引用数据类型，统一是将 盒子中保存的那个值复制一份，传递下去。

              # 构造方法

              • 构造方法是一个比较特殊的方法，通过构造方法可以完成对象的创建，以及实例的初始化，换句话说 构造方法是用来创建对象的，并且同时给对象的属性赋值。
              • 当一个类没有提供任何构造方法，系统会默认提供一个无参数的构造方法，而这个构造方法被称为缺省构造器。
              • 构造方法采用new运算符来调用构造方法：new 构造方法名()
              - 构造方法的语法结构：
[修饰符列表] 构造方法名 (形式参数列表){
  构造方法体
}
注意：修饰符列表统一写成public
构造方法名和类名必须一致
构造方法不需要指定返回值类型，不能为void
通常构造方法体中给属性赋值，完成属性的初始化
当实例变量没有手动赋值时，系统会自动赋值，且是在构造函数方法执行的过程中给属性赋值

              # 实例方法

              • 普通方法：带有static；调用：类名.方法名
              • 实例方法：不带有static；调用：先创建对象（通过构造方法创建对象），引用.实例方法名

              # 封装

              • 作用：保证内部结构安全，屏蔽复杂，暴露简单
              • 属性私有化，且对外提供公开的get和set方法作为操作入口，并且都不带static，都是实例方法
              //封装get,set 方法

  private int age;
  public int getAge(){
    return age;

  public void setAge(int nianling){
    //可以设置一个关口用来判断
    if(nianling<0||nianling>150){
      System.out.prientln("年龄不合法")
      return;
    }
    age = nianling
  }
}

              # this和static

              • static 静态，所有static关键字修饰的都是类相关的
              • static修饰的都采用 类名. 的方式访问
              • static修饰的变量：静态变量
              • static修饰的方法：静态方法
              • 不带static是与实例相关的，访问采用“引用.”的方式，需要先new对象

              # 什么时候声明实例对象，什么时候声明静态变量

              • 如果这个类的所有对象的某个属性是一样的，不建议定义成实例对象，浪费内存，建议使用静态变量。
              • 静态变量在类加载时初始化，不需要new对象静态变量方法就有了，存储在方法区。
              • 空引用访问静态变量不会导致空指针。

              # 什么时候定义为实例方法，什么时候定义为静态方法

              • 类 = 属性+方法
              • 1、静态方法属于整个类所有，因此调用它不需要实例化，可以直接调用（类.静态方法（））。 实例方法必须先实例化，创建一个对象，才能进行调用（对象.实例方法（））。
              • 2、静态方法只能访问静态成员，不能访问实例成员；而实例方法可以访问静态成员和实例成员。
              • 3、在程序运行期间，静态方法是一直存放在内存中，因此调用速度快，但是却占用内存。 实例方法是使用完成后由回收机制自动进行回收，下次再使用必须再实例化。
              • 4、一般来说，公共的函数、经常调用的可以写成静态方法，比如数据连接等（SqlHelper)。
