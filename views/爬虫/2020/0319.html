                          爬虫框架之Scrapy

                          juverson 2020-03-19 爬虫

                          # 理解Scrapy

                          • Scrapy 是框架
                          • 采用异步框架，实现高效的网络采集
                          • 最强大的框架

                          # 安装

                          pip install scrapy

                          # scrapy主要包括以下组件

                          • 引擎(Scrapy)用来处理整个系统数据流，触发事务(框架核心)
                          • 调度器(Scheduler)用来接受引擎发过来的请求，压入队列中，并在引擎再次请求时返回，可以想象成一个URL的优先队列，由它决定下一个要抓取的网址，同时去掉重复网址
                          • 下载器(Downloader)用于下载网页内容，并将网页内容返回给Scrapy，下载器是建立在twisted这个高效的异步模型之上
                          • 爬虫(Spiders)爬虫主要是干活的，用于从特定的网页中提取自己所需要的信息，即所谓的实体，用户也可以从中提取链接让Scrapy继续抓取下一个链接
                          • 项目管道(Pipeline)负责处理爬虫从网页中抓取的实体，主要的功能是持久化实例，验证实例的有效性，清楚不需要的信息。当页面被爬虫解析后，将被发送到项目管道，并经过特定的次序处理数据
                          • 下载器中间件(Downloader Middlewares)位于Scrapy引擎和下载器之间的框架，主要处理Scrapy引擎与下载器之间的请求及响应
                          • 爬虫中间件(Spider Middlewares)介于Scrapy引擎和爬虫之间的框架，主要工作是处理爬虫的响应输入和请求输出
                          • 调度中间件(Scheduler Middlewares)介于Scrapy引擎和调度器之间的中间件，从Scrapy引擎发送到调度的请求和响应

                          # 运行流程如下

                          • 引擎从调度器中提取出一个链接(URL)用于接下来的抓取
                          • 引擎把URL封装成一个请求(Request)传给下载器
                          • 下载器把资源下载下来，并封装成应答包(Response)
                          • 爬虫解析Response
                          • 解析出实体，则交给实体管道进行进一步的处理
                          • 解析出的是链接(URL),则把URL交给调度器等待抓取

                          # 创建项目

                          scrapy startproject demo_name cd demo_name

                          # 创建爬虫

                          scrapy genspider 爬虫名字 域名(你要爬的网站 xxx.com) 注意爬虫名字不能与项目名字一样必须唯一 网站域名是允许爬虫采集的域名

                          # 提取数据

                          • 正则表达式(基础，必会，难掌握)
                          • XPath -->从HTML中提取数据语法

                          # Selectors选择器

                          Scrapy提取数据有自己的套机制。他们被称为选择器(seletors)，因为他们通过特定的XPath或CSS表达式选择HTML文件中的某个部分。本身是基于parsel模块。

                          • 我们可以通过使用浏览器的开发者工具来得到我们所需要的节点的XPath表达式，然后通过xpath()方法来得到其内容
                          • response.xpath('xpath表达式').getall() ---->或者.get()
                          • xpath语法

                          # 运行爬虫

                          scrapy crawl 爬虫名

                          # 反爬虫

                          • custom_setting
                          • 在settings.py中去掉DEFAULT_REQUEST_HEADERS(42)的注释，并且加上一个User-Agent

                          # 保存数据

                          scrapy crawl 爬虫名 -o ip.csv(生成文件名及格式)

                          # 例子

                          # -*- coding: utf-8 -*-
import scrapy


class XicidailiSpider(scrapy.Spider):
    name = 'xicidaili'
    allowed_domains = ['xicidaili.com']
    start_urls = ['https://xicidaili.com/nn/5']

#解析响应数据，提取数据或者网址，response就是网页源码
    def parse(self, response):  
        #提取数据
        #提取ip port 
        selectors = response.xpath('//tr') #选择所有的tr标签
        # 循环遍历tr标签下的td标签
        for selector in selectors:
            ip = selector.xpath('./td[2]/text()').getall() #当前节点下继续选择第二个td标签
            port = selector.xpath('./td[2]/text()').getall() #当前节点下继续选择san个td标签
            #使用一下方法保存数据
            items = {
                'ip' : ip,
                'port': port
			}
            yield items

        # 翻页操作
        next_page = response.xpath('//a[@class="next_page"]/@href').get()
        if next_page:
            next_url = response.urljoin(next_page)
            #发出请求 yield是生成器 callback是回调函数，将请求得到的响应叫给自己处理，parse是最上面定义的函数
            yield scrapy.Request(next_url, callback=self.parse)
