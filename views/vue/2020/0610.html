                                  # es6数组方法的使用forEach,some,filter

                                  • forEach:伪数组转成真数组才能遍历
                                  <div id="app">
    <ul ref="ulParent">
	//ref 是vue封装好的元素，绑定ref之后，this.refs会把所以绑定ref的值存在一起，可以直接去取
        <li style="height:60px">1</li>
        <li style="height:160px">2</li>
    </ul>
    <button @click="getAllLiHeight">获取高度数组</button>
</div>
<script>
new Vue({
	el:'#app',
	methods:{
		getAllLiHeight(){
			let liHeightArr = []
			//1.获取dom元素
			let allLis = this.$refs.ulParent.getElementsByTagName('li');
			//2.伪数组转化成真数组
			Array.prototype.slice.call(allLis).forEach(li =>{
				liHeughtArr.push(li.clientHeight);
			})
		}
	}
})
</script>

                                  # localStorage:

                                  • 生命周期是永久，这就意味着除非用户显示在浏览器提供的UI上清除localstorage信息，否则这些信息永久存在

                                  # 常用本低localstorageAPI:

                                  • localStorage.setItem('key','value') 以key为名称存储一个值value
                                  • localStorage.getItem('key') 获取名为KEY的值
                                  • localStorage.removeItem('key') 删除名称为key的信息
                                  • localStorage.clear() 清除localStorage中所有信息

                                  # 在本地存储的都是非关系行的数据库，一般都是，键，值，且都是字符串，而在页面是我们都到的是对象，所以

                                  就有一个转化的过程，就用到JSON,JSON.parse转化成对象，JSON.stringfy转化成字符串

                                  # 事件修饰符

                                  • .stop 阻止冒泡，从内到外,是真正意义上的阻止冒泡
                                  • .prevent 组织默认事件
                                  • .capture 添加事件侦听器时使用事件捕获模式，从外到内
                                  • .self 只当事件在该元素本身（比如不是子元素）触发时触发回调，只会阻止自己身上冒泡行为的触发，并不会真正阻止冒泡行为
                                  • .once 事件只触发一次
